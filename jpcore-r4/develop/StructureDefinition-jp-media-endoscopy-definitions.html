Resource Profile: JP_Media_Endoscopy - Detailed Descriptions

Active as of 2023-04-21

Definitions for the jp-media-endoscopy resource profile.

  • Differential Elements Table
  • Key Elements Table
  • Snapshot Elements Table

Guidance on how to interpret the contents of this table can be found here.

1. Media
Definition

内視鏡検査で取得または使用される画像、映像、音声に関わる情報。

「ヘルスケアで取得または使用される写真、ビデオ、またはオーディオ録音。実際の内容はインラインで提供されるか、直接参照によって提供される場合があります。」

2. Media.identifier
Definition

製品またはシステムが管理する、施設内で画像（JPEG等）、映像（MPEG等）、音声ファイルなどのメディアを一意に識別するためのID。

画像に関連付けられた識別子 - これには、画像自体の識別子、その収集の文脈（シリーズIDなど）の識別子、および収集工程の識別子としてのアクセス番号などの文脈IDが含まれる場合があります。

NoteThis is a business identifier, not a resource identifier (see discussion)
Must Supporttrue
3. Media.basedOn
Definition

他のシステムから依頼されたオーダ情報。

このメディアの作成によって完全または一部実行される手順。

Comments

通常、依頼元となるServiceRequestリソースを参照する。他のシステムと連携していない場合は参照不要。

4. Media.partOf
Definition

参照されるイベントの一部。

この特定のイベントが構成要素またはステップである、より大きなイベント。

Comments

使用する場合は、基底にある Definition、Requirement、Comments の内容を参考に使用する。

「イベントを出会いにリンクするために使用しないでください。それにはMedia.encounterを使用してください。 [許可された参照リソースは、適切にイベントリソースに合わせて調整される場合があります]。」

5. Media.status
Definition

メディアのステータス。

「{{title}}の現状。」

Comments

”completed” を指定する。

「[[event.html#statemachine | イベントパターン]]ドキュメントには名目状態遷移図があります。」 「未知は「その他」を表さない。定義されたステータスのいずれかが適用される必要がある。著者システムが現在の状態を確信できない場合には未知を使用する。」

6. Media.type
Definition

メディアの種類。

「メディアが画像、動画、音声録音、またはその他のメディアカテゴリーであるかどうかを分類するコード。」

Comments

使用する場合は、メディアの種類を image, video, audio から選択する。

7. Media.modality
Definition

メディアを取得・撮影した装置（モダリティ）。

メディアの種類の詳細 - 通常はどのようなデバイスから取得されたか（どのタイプのデバイスか）。 DICOMシステムから取得された画像がMediaリソースでラップされている場合、これがモダリティです。

Comments

使用する場合、ImagingStudy同様”ES”を指定する。

8. Media.view
Definition

メディアのイメージングビュー（例：横方向、前後方向など）。

画像表示の名称。例えば、側面視画像や前後方向（AP）視画像。

Comments

内視鏡では省略してよい。将来的にニーズが出てきた場合には検討する。

9. Media.subject

メディアの対象患者に関する情報。

TypeReference(JP_Patient|Patient|Practitioner|PractitionerRole|Group|Device|Specimen|Location)

10. Media.encounter

TypeReference(JP_Encounter|Encounter)

これは通常、メディアが発生した出会いです。 (Kore wa tsūjō, media ga hassei shita deai desu.)

12. Media.issued

20. Media.frames
Definition

フレーム数。

１枚の写真にあるフレーム数。これは、複数ページのファックスや、単一のイメージで複数のスライスを取得する画像取得コンテキスト、またはアニメーション GIF で使用されます。複数のフレームがある場合、これは値を持たなければならず、マルチフレーム対応のレンダリングウィジェットが必要であることをインターフェースソフトウェアに警告するためです。

Comments

EUSの場合など、フレーム数が1より大きいメディア（単一のメディアで複数のスライスを取得する画像取得コンテキスト）を扱う場合に使用する。

フレーム数が提供されていない場合、値は不明かもしれません。応用プログラムは、明示的に指定されていない限り、1つのフレームしか存在しないと仮定してはいけません。

21. Media.duration
Definition

ビデオ映像・音声向けの、秒単位の録画・記録時間。

レコーディングの時間（音声、ビデオとも）は秒単位で表記してください。

22. Media.content

埋め込まれたメディアの実際のコンテンツ、またはメディアソースファイルへの直接参照。

メディアの実際のコンテンツ - インラインまたはメディアソースファイルへの直接参照による。
